<objective>
Build a pure encoder service that converts a rendered `CGImage` plus source metadata plus `ExportOptions` into encoded image `Data` using `CGImageDestination` (NOT `UIImage.jpegData`). This is the heart of Phase 5: it owns format selection, quality, color-profile embedding, EXIF preservation, GPS stripping, and resize-to-target.

Purpose: Centralizing encoding here means the view-model just calls `ExportService.encode(...)` and gets ready-to-save bytes. The encoder is testable in isolation and is the SINGLE place the GPS-strip and EXIF-preserve rules live (PITFALL #16).
Output: `PhotoEditor/Export/ExportService.swift` with one public entry point and private helpers for resize / metadata sanitization / properties dictionary construction.
</objective>

  <behavior>
    - Public API:
      `enum ExportService {
         enum Error: Swift.Error { case encodeFailed, unsupportedFormat, resizeFailed }
         static func encode(
             cgImage: CGImage,
             sourceProperties: [CFString: Any],
             colorSpace: CGColorSpace?,
             options: ExportOptions
         ) throws -> Data
       }`
    - Resize behavior: derive sourceLongEdge from cgImage.width/height; call options.size.resolve(sourceLongEdge:); if target != source, downsample using a CIContext + Lanczos transform; otherwise pass through. Resize NEVER upscales beyond source (resolve already clamps but extra guard: if target > sourceLongEdge in .full case it's by definition equal).
    - JPEG: destination type = UTType.jpeg.identifier; properties include kCGImageDestinationLossyCompressionQuality = options.quality.
    - HEIC: destination type = UTType.heic.identifier; quality key set. If CGImageDestinationCreateWithData returns nil for HEIC (older simulator without HEIC encoder), fall back to JPEG and proceed.
    - PNG: destination type = UTType.png.identifier; quality key OMITTED.
    - Color space: if colorSpace is non-nil, set on the destination via the properties dict's `kCGImagePropertyProfileName` is wrong — instead pass the colorSpace by tagging the CGImage itself before adding (cgImage.copy(colorSpace:)). The destination embeds whatever profile the CGImage carries.
    - EXIF preservation: copy the following keys from sourceProperties into the per-image properties dict if present:
      kCGImagePropertyTIFFDictionary, kCGImagePropertyExifDictionary
    - GPS / IPTC strip: even if sourceProperties contains them, DO NOT copy:
      kCGImagePropertyGPSDictionary, kCGImagePropertyIPTCDictionary
    - Orientation: set kCGImagePropertyOrientation = 1 explicitly in properties dict (we baked rotation in ImageImporter; downstream tools must see Up). Also overwrite TIFF dict's orientation key to 1 if a TIFF dict was carried.
    - On any CGImageDestinationCreateWithData / CGImageDestinationFinalize failure: throw .encodeFailed.
    - The encoder is non-isolated (free function on enum). Caller is responsible for off-main dispatch.
  </behavior>
  <action>
    Create `PhotoEditor/Export/ExportService.swift`.

    Imports: `import Foundation`, `import ImageIO`, `import CoreGraphics`, `import CoreImage`, `import UniformTypeIdentifiers`.

    Step-by-step inside `encode`:
    1. Tag the CGImage with the source colorSpace (or fall back to displayP3 then sRGB):
       ```swift
       let outputCS = colorSpace ?? CGColorSpace(name: CGColorSpace.displayP3) ?? CGColorSpace(name: CGColorSpace.sRGB)!
       guard let tagged = cgImage.copy(colorSpace: outputCS) else { throw Error.encodeFailed }
       ```
       Per PITFALL #4: never use CGColorSpaceCreateDeviceRGB. Use named profiles. Honor source profile preference (P3 in → P3 out).
    2. Resize tagged → resized via CIImage(cgImage: tagged).transformed(by: CGAffineTransform(scaleX: s, y: s)) and rendering with a fresh CIContext that has workingColorSpace = extendedLinearSRGB and outputColorSpace = outputCS. Skip resize when scale == 1.0 (within 0.5 px tolerance).
    3. Resolve UTI:
       - Start with options.format.uti
       - If format == .heic: probe with `CGImageDestinationCreateWithData(NSMutableData() as CFMutableData, UTType.heic.identifier as CFString, 1, nil)`; if nil, fall back to UTType.jpeg.identifier. (Per CONTEXT decisions: "fallback to JPEG if device doesn't support".)
    4. Build properties dict:
       ```swift
       var props: [CFString: Any] = [
         kCGImagePropertyOrientation: 1
       ]
       if format.supportsQuality {
         props[kCGImageDestinationLossyCompressionQuality] = options.quality
       }
       // EXIF passthrough
       if let tiff = sourceProperties[kCGImagePropertyTIFFDictionary] as? [CFString: Any] {
         var sanitizedTiff = tiff
         sanitizedTiff[kCGImagePropertyTIFFOrientation] = 1
         props[kCGImagePropertyTIFFDictionary] = sanitizedTiff
       }
       if let exif = sourceProperties[kCGImagePropertyExifDictionary] as? [CFString: Any] {
         props[kCGImagePropertyExifDictionary] = exif
       }
       // NOTE: deliberately NOT copying kCGImagePropertyGPSDictionary or kCGImagePropertyIPTCDictionary.
       ```
    5. Create destination, add image, finalize:
       ```swift
       let data = NSMutableData()
       guard let dest = CGImageDestinationCreateWithData(data as CFMutableData, uti as CFString, 1, nil) else {
         throw Error.encodeFailed
       }
       CGImageDestinationAddImage(dest, resizedCG, props as CFDictionary)
       guard CGImageDestinationFinalize(dest) else { throw Error.encodeFailed }
       return data as Data
       ```

    Pitfalls explicitly addressed (cite in code comments):
    - PITFALL #16: avoid UIImage round-trip; CGImageDestination preserves color profile + lets us control metadata.
    - PITFALL #4: named color spaces only; honor source profile.
    - PITFALL #3: orientation is baked at import; we still set kCGImagePropertyOrientation=1 so downstream tools don't re-rotate.

    DO NOT call PHPhotoLibrary or UIActivityViewController here — encoding is pure. Saving is a separate concern (plan 05-03).
  </action>
