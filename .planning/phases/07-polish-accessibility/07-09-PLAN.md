For each file, read it and apply the following sweep rules:

**Replacement rules:**
- `.font(.headline)` on Text → `.font(Theme.Typography.subtitle)`
- `.font(.subheadline)` on Text → `.font(Theme.Typography.subtitle)`
- `.font(.subheadline.weight(.medium))` on Text → `.font(Theme.Typography.subtitle)`
- `.font(.body)` on Text → `.font(Theme.Typography.body)`
- `.font(.caption)` on Text → `.font(Theme.Typography.caption)`
- `.font(.caption.monospacedDigit())` on Text (numeric value) → `.font(Theme.Typography.valueBubble)`
- `.font(.caption2)` on Text → `.font(Theme.Typography.caption)`
- `.font(.title)` / `.title2` / `.title3` on Text → `.font(Theme.Typography.title)`
- `.font(.system(size: NN, weight: ..., design: ...))` on Text labels → choose closest Theme.Typography role.

**KEEP (do NOT change):**
- `.font(.system(size: NN, weight: .semibold))` applied to `Image(systemName:)` — SF Symbol icon sizing.
- `.font(.system(size: 44, weight: .semibold))` on empty-state icons — also icon sizing.
- Empty-state Text inside library/recipes views that already uses `.font(.headline)` etc. — these are covered by the rule above and should be replaced.

**Frame audit:**
- If you find a Text with `.frame(width: NN)` where NN < 80, REMOVE the fixed width (it will truncate at XL Dynamic Type).
- Tab bar `frame(width: 64)` for icons + label is acceptable IF icons stay at the fixed font size and label uses Theme.Typography.caption — Dynamic Type still respects line wrapping.

For each file: read it, identify the lines matching the patterns above, and Edit them. List in the SUMMARY which lines per file you changed.

**LibraryItemThumbnail.swift specifically:** likely shows a recipe-name or date label. Apply the same rules.
