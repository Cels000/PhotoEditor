Read the current `CurvesPanelView.swift`. Find the view that renders each draggable curve point (likely a Circle / shape inside a ForEach). For each point view, append:

```swift
.accessibilityElement()
.accessibilityLabel("Curve point \(index + 1)")
.accessibilityValue("Y value \(Int(point.y * 100)) percent")
.accessibilityAdjustableAction { direction in
    let step = 0.05
    var p = points[index]
    switch direction {
    case .increment: p.y = min(1.0, p.y + step)
    case .decrement: p.y = max(0.0, p.y - step)
    @unknown default: break
    }
    points[index] = p
    // Trigger viewModel/binding update — match existing onChanged path
}
```

- Each option: `.accessibilityLabel("<channel> channel")`, `.accessibilityAddTraits(isSelected ? .isSelected : [])`, `.accessibilityHint("Switches the curve to this channel")`.

For the curve canvas itself: `.accessibilityLabel("Tone curve")`, `.accessibilityHint("Drag points to shape the curve. Use VoiceOver swipe up/down on each point to adjust.")`.

Notes:
- DO NOT change the existing drag gesture math.
- If the existing view structure makes wrapping each point cumbersome, use `.accessibilityRotorEntries` as fallback OR add an `.accessibilityChildren { ... }` block listing each point as a child element. Whichever fits the existing structure cleanly.
- READ THE FILE FIRST. Adapt the snippet to existing variable names (points array, binding pattern).
