# Phase 4: Library + Persistence - Context

**Gathered:** 2026-05-03
**Status:** Ready for planning

<domain>
## Phase Boundary

Add an in-app library of edited photos. Edits persist across app launches; users can re-open any item and continue editing exactly where they left off. SwiftData with `VersionedSchema` from day one (per PITFALLS guidance ‚Äî adding it after data ships is painful).

Specifically:
- `LibraryItem` SwiftData model: id (UUID), createdAt, updatedAt, source PHAsset localIdentifier, adjustment-stack JSON blob, thumbnail Data
- `LibraryStore` service: save/update/delete/fetch, thumbnail generation, observation
- `LibraryGridView`: grid of thumbnails with "edited" badge (always shown ‚Äî every library item is by definition edited), sort newest-first
- Re-edit flow: tap library item ‚Üí reopen editor with stored stack restored
- Delete flow: confirmation alert, removes thumbnail blob too
- Graceful PHAsset deletion handling: if source is gone, show fallback placeholder + read-only mode (or option to remove from library)
- Permission handling: `.limited` access ‚Äî UX explains and degrades gracefully

</domain>

<decisions>
## Implementation Decisions

### Claude's Discretion

- **Persistence:** SwiftData (iOS 17+). `VersionedSchema` from v1 ‚Äî even if v1 has only one schema. Migration plan stub for future.
- **Schema (v1):**
  ```
  @Model final class LibraryItem {
      var id: UUID
      var createdAt: Date
      var updatedAt: Date
      var sourceAssetID: String?  // PHAsset localIdentifier; nil if PHAsset gone
      var stackData: Data         // JSON-encoded AdjustmentStack
      var thumbnailData: Data?    // 400√ó400 JPEG
      var schemaVersion: Int      // mirrors AdjustmentStack.schemaVersion
  }
  ```
- **JSON blob, not normalized columns:** AdjustmentStack changes are field-additive (new sliders, new filters). JSON Codable round-trips with default values handle additions for free; only renames need migration.
- **LibraryStore service:** `@Observable final class`. Owns `ModelContainer` (built at app init), exposes `save(stack:source:thumbnail:)`, `update(item:stack:)`, `delete(_:)`, `items: [LibraryItem]` via `@Query` (or refresh-on-write).
- **Thumbnail generation:** 400√ó400 px JPEG `.6` quality, generated by `RenderEngine.previewContext.createCGImage` after applying current stack at 400px long-edge. ~30 KB each. Done off-main on `Task.detached(.background)`.
- **Save flow integration:** `EditorViewModel.saveToLibrary()` ‚Äî generates thumbnail + persists. Different from "Save to Photos" (Phase 5 export). Replaces or supplements current save behavior.
- **Re-edit flow:** Tapping library item calls `EditorViewModel.openLibraryItem(_ item: LibraryItem)`. ImageImporter loads the PHAsset (if available), `stackData` decodes back to AdjustmentStack, undo stack reset.
- **Library entry point:** Top-left toolbar button ("Library" / SF Symbol `photo.on.rectangle.angled`) ‚Üí sheet with grid. Selecting an item dismisses sheet and loads it.
- **PHAsset deleted handling:** When `PHAsset.fetchAssets(withLocalIdentifiers: [id])` is empty, show "Source no longer in Photos" placeholder. Option to delete the library item.
- **Permission scope:** Already declared in Info.plist. `.limited` access ‚Äî when picking, show standard limited-library prompt. When library item references an asset outside the limited set, treat as deleted and show placeholder.
- **Deletion confirmation:** `.alert` with "Delete edit?" / "Cancel"+"Delete" (destructive style).
- **Sort:** updatedAt DESC. Edited-most-recently appears first.

</decisions>

<specifics>
## Specific Ideas

- LibraryItem references `sourceAssetID: String?` ‚Äî NOT a binary copy of the source. If user deletes source from Photos, we lose the editable original (by design, matches "non-destructive editing of user's photos, not in-app duplication").
- Optional v2 enhancement: store a 2048px JPEG backup of the source so deletions don't break library items. NOT in v1 ‚Äî too much storage cost without user request.
- `LibraryGridView` should use `LazyVGrid` 3-column. Thumbnail aspect: square (cropped center).

</specifics>
